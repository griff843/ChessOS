# M12D - Cognitive Training Contract

## Overview

M12D extends Chess-OS from tactical-only training into deterministic multi-modal cognitive training.

Supported exercise types:
- `tactical`
- `recall`
- `visualization`
- `reconstruction`

All selection, generation, and grading logic is canonical in `packages/training/src/cognitive` and shared engine layers.

## Deterministic Session Composition

1. Mixed sessions are generated by `buildMixedSession(...)`.
2. Type mix is deterministic from canonical objective/mix input.
3. If a requested type lacks eligible positions, the session degrades to tactical slots and records reasons in `mix-rationale.json`.
4. Every mixed session writes:
- `out/sessions/<sessionId>/study-session.json`
- `out/sessions/<sessionId>/cognitive-exercises.json`
- `out/sessions/<sessionId>/mix-rationale.json`

## Exercise Contracts

### Recall

Flow:
1. Timed viewing window.
2. Board hidden.
3. User reconstructs pieces.

Grading output fields:
- `totalPieces`
- `correctPieces`
- `incorrectPieces`
- `missingPieces`
- `extraPieces`
- `accuracy`
- `gradingTier`
- `isCorrect`

### Visualization

Flow:
1. Show move-sequence prompt without persistent board context.
2. Ask deterministic question (`final_square`, `piece_count`, `check_status`, `king_location`, `material_balance`).

Grading output fields:
- `isCorrect`
- `userAnswer`
- `correctAnswer`
- `questionType`
- `gradingTier`

### Reconstruction

Flow:
1. Position from game corpus.
2. User chooses continuation.

Grading output fields:
- `userMove`
- `gameMove`
- `engineMove`
- `isExactGameMove`
- `isEngineMove`
- `gradingTier`
- `evalLossCp`
- `isCorrect`

## Pattern Library Contract

Pattern categories:
- `fork`
- `pin`
- `skewer`
- `back_rank`
- `clearance`
- `deflection`
- `king_attack`
- `endgame_technique`
- `unclassified`

`pattern-library.json` entry fields include:
- `pattern`
- `totalSeen`
- `totalCorrect`
- `accuracy`
- `missRate`
- `severity`
- `trendDirection`
- `recentAccuracy`
- `improvementTrend`
- `difficultyDistribution`
- `exampleExerciseIds`

## Explainability Requirements

Each cognitive exercise record must include:
- `exerciseType`
- `rationale`
- `patternCategory` (when applicable)
- mode-specific data payload (`recallData`, `visualizationData`, `reconstructionData`, `tacticalData`)

No hidden grading heuristics are permitted.
